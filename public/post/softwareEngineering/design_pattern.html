Design Pattern

Date: 2022-05-21. Last updated: 2023-04-03.

  1. Generalization 泛化
    继承，一般与特殊的关系
    實線三角箭頭，箭頭指向父類
  2. Realization 實現
    类与接口，类是接口所有特征和行为的实现
    虛線三角箭頭，箭頭指向接口
  3. Association 關聯
    擁有，壹個類知道另壹個類的屬性和方法
    實線 普通箭頭，箭頭指向被擁有者
    該對象持有另壹個對象的指針或引用
  4. Dependency 依賴
    使用
    虛線 普通箭頭 箭頭指向被使用者
    局部變量、方法的參數或者對靜態方法的調用
  5. Composition 組合
    整體與部分 部分不可以離開整體而單獨存在
    實心菱形實線 菱形指向整體 成員變量
  6. Aggregation 聚合
    整體與部分 部分可以離開整體而單獨存在
    空心菱形實線 菱形指向整體
    存在壹個容器包含別壹個類

泛化 實現 三角箭頭
關聯 依賴 普通箭頭
組合 聚合 菱形

面向對象設計原則

Name Definition
单一职责原则 一个类应该有且仅有一个引起它变化的原因
开闭原则 对扩展开放，而对修改关闭
依赖倒转原则 对抽象编程，不要对实现编程
接口隔离原则 Dooley
合成复用原则 Dooley
迪米特原则 不要和陌生说话

六大设计原则:

  1. 依赖倒置
  2. 接口隔离
  3. 最少知识
  4. 单一职责
  5. 开放封闭
  6. 里氏替换
补充设计原则：
  1. 组合/聚合复用
  2. 无环依赖
  3. 共同封装
  4. 共同重用
  5. 好莱坞原则
