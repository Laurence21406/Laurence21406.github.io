Design Pattern

Date: 2022-05-21. Last updated: 2023-04-03.

  1. Generalization 泛化
    继承，一般与特殊的关系
    三角箭头实线，箭头指向父类
  2. Realization 實現
    类与接口，类是接口所有特征和行为的实现
    三角箭头虚线，箭头指向接口
  3. Association 關聯
    拥有，一个类知道另一个类的属性和方法
    普通箭头实线，箭头指向被拥有者
    该对象持有另一个对象的指针或引用
  4. Dependency 依賴
    使用
    普通箭头虚线 箭头指向被使用者
    局部变量、方法的参数或者对静态方法的调用
  5. Aggregation 聚合
    整体与部分 部分可以离开整体而单独存在
    空心菱形实线 菱形指向整体
    存在一个容器包含别一个类
  6. Composition 組合
    整体与部分 部分不可以离开整体而单独存在
    实心菱形实线 菱形指向整体 成员变量

泛化 實現 三角箭頭
關聯 依賴 普通箭頭
聚合 組合 菱形

面向对象设计原则

Name Definition
单一职责原则 一个类应该有且仅有一个引起它变化的原因
开闭原则 对扩展开放，而对修改关闭
依赖倒转原则 对抽象编程，不要对实现编程
接口隔离原则 Dooley
合成复用原则 Dooley
迪米特原则 不要和陌生说话

六大设计原则:

  1. 依赖倒置
  2. 接口隔离
  3. 最少知识
  4. 单一职责
  5. 开放封闭
  6. 里氏替换
补充设计原则：
  1. 组合/聚合复用
  2. 无环依赖
  3. 共同封装
  4. 共同重用
  5. 好莱坞原则
